# Security policy

## Reporting a vulnerability

If you find a security issue in the **website code** or the **release
pipeline**, please report it privately to `security@opendomaindata.org`. We
will respond within 5 business days and credit reporters in the changelog on
request.

Please do **not** open a public GitHub issue for security reports.

## Out of scope

- Data correctness issues — those are not security issues. Use the
  [correction template](.github/ISSUE_TEMPLATE/correction.md).
- Findings against `*.opendomaindata.org` infrastructure that don't affect
  the open dataset or the website code (e.g. CDN configuration).

## Supported versions

The project supports the latest tagged release and `main`. Older dataset
releases remain available for citation but do not receive backports.
